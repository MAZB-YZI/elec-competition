## 一、硬件资源分配（A板）

| 功能 | 引脚 | CubeMX 外设 | 说明 |
|---|---|---|---|
| K1 按键 | PA0 | GPIO Input Pull-Down | 板载 WKUP，按下=高电平 |
| D1 呼吸灯 | PB0 | TIM3_CH3 PWM | 板载 LED2(PB2)无PWM，外接LED到PB0 |
| Vi 信号采集 | PC0 | ADC1 IN10 + DMA | 外部输入 0~3.3V |
| ADC 触发时钟 | 无引脚 | TIM2 TRGO | 1kHz 定时触发 ADC |
| 1ms 系统节拍 | 无引脚 | TIM6 中断 | 按键扫描 + 呼吸灯步进 |
| 100ms 刷新 | 无引脚 | TIM7 中断 | 显示刷新 + 心跳发送 |
| 与B板通信 | PA9(TX) / PA10(RX) | USART1 DMA | 115200，交叉连接B板 |
| OLED 显示 | PB6(SCL) / PB7(SDA) | I2C1 | SSD1306，地址 0x78 |

## 二、A板需要实现的功能清单

### 2.1 Vi 信号采集与识别（对应评分项 2、3、11、12、13、14）

**ADC 配置：**
- ADC1 IN10（PC0），连续采样，DMA Circular，TIM2_TRGO 触发，采样率 1kHz
- 采样时间选 480 Cycles（最长，提高精度）

**信号识别算法（1s 窗口，1000个点）：**

```c
// 计算 max、min、均值
float vi_max, vi_min, vi_avg;

// 判断逻辑：
// vi_max - vi_min < 0.05V  且  vi_avg < 0.1V  → 无输入
// vi_max - vi_min < 0.05V  且  vi_avg >= 0.1V → 直流，显示均值
// 过零次数 2~6次/秒                            → 正弦波，显示峰值(max-1.65V)
// 过零次数 > 10次/秒                           → 方波，显示幅值(max-1.65V)
```

**显示格式：**
```
SigType: SINE        (或 DC / SQUARE / NONE)
Vi:  1.65V
Vmax:2.30V
Vmin:1.00V
```

### 2.2 D1 呼吸灯（对应评分项 8、9、10）

- 使用 TIM3_CH3（PB0）输出 PWM 驱动 D1
- 呼吸效果：CCR 按正弦曲线 0→ARR→0 循环变化
- **三种周期由 B板 PWM 模式决定**，从串口接收后实时切换：

| B板 pwm_mode | D1 呼吸周期 | 步进间隔 |
|---|---|---|
| 0（跟随） | T1 = 0.5s | 每 1ms 步进，半周期 250步 |
| 1（最大值） | T2 = 1s | 每 1ms 步进，半周期 500步 |
| 2（最小值） | T3 = 2s | 每 1ms 步进，半周期 1000步 |

- K1 短按：切换 D1 开/关（`d1_enable` 标志）
- D1 点亮时立即使用当前最新的 `pwm_mode` 对应周期，无过渡

### 2.3 K1 按键处理（对应评分项 4、5）

在 TIM6 1ms 中断里扫描：

```c
// 短按（20ms ~ 600ms）：切换 D1 开关
// 长按（>= 600ms）：发送 D2_CMD_TOGGLE 给 B板，控制 D2 熄灭/点亮
//                   失联期间长按：暂存 d2_cmd_pending，复联后发送
```

### 2.4 显示内容（对应评分项 1、7）

OLED 128×64，8行，调用 `SSD1306_DrawString(page, col, str)`：

```
Page0: A Board    [OK]     ← 链路状态
Page1: SigType:SINE        ← 信号类型
Page2: Vi:  1.65V          ← 实时电压
Page3: Vmax:2.30V          ← 1s最大值
Page4: Vmin:1.00V          ← 1s最小值  
Page5: D1:ON  D2:OFF       ← 两板LED状态
Page6: MODE:FOLLOW         ← B板当前PWM模式
Page7: (空)
```

### 2.5 通信（对应评分项 23~30）

**心跳发送（每 200ms 一次，TIM7 触发）：**

```c
Frame_Pack(&f,
    SRC_A,
    d1_enable,          // 本板 D1 状态
    0,                  // A板无 D2，填 0
    0,                  // pwm_mode A板不填，填 0
    d2_cmd              // 平时填 D2_CMD_NONE(0)
                        // K1长按时填 D2_CMD_TOGGLE(1)，发一次后清零
);
```

**接收解析（UART IDLE 中断）：**

```c
// 收到 B板帧（src == SRC_B）后：
peer_d2_on   = frame.d2_on;       // 更新B板D2状态，用于显示
peer_pwm_mode = frame.pwm_mode;   // 更新呼吸周期
last_rx_tick  = sys_ms;           // 刷新链路时间戳
```

**失联检测：**
```c
// 超过 800ms 未收到有效帧 → link_ok = 0，显示 [FAIL]
// 收到第一帧后立即 → link_ok = 1，显示 [OK]
// 失联期间 D1 保持当前呼吸，不更新 pwm_mode
```

## 三、CubeMX 配置要点（A板）

| 外设 | 关键参数 |
|---|---|
| TIM2 | PSC=83, ARR=999, TRGO=Update Event（1kHz ADC触发） |
| TIM3 CH3 | PSC=0, ARR=839, CCR初始=0（100kHz PWM，D1呼吸） |
| TIM6 | PSC=83, ARR=999，开NVIC中断（1ms） |
| TIM7 | PSC=8399, ARR=999，开NVIC中断（100ms） |
| ADC1 | IN10(PC0), Continuous, DMA Circular, Trigger=TIM2_TRGO, Sample=480Cycles |
| USART1 | 115200, DMA RX Circular, 开NVIC Global Interrupt |
| I2C1 | Standard Mode 100kHz |
| PA0 | GPIO Input Pull-Down（K1按键） |
| PB0 | TIM3_CH3复用（D1 PWM） |

## 四、与B板的连线方式

```
A板                    B板
PA9  (USART1_TX) ───→─── PA10 (USART1_RX)
PA10 (USART1_RX) ───←─── PA9  (USART1_TX)
GND              ─────── GND
```

> 共 3 根线，满足题目"含地线不超过3根"要求。

## 六、联调顺序建议

1. **各自先跑通单板功能**
   - A板：ADC 采样数值正确，D1 呼吸灯三种频率可切换
   - B板：示波器确认 PE8/PE9 有 5kHz 互补 PWM 输出，死区可见

2. **串口先用串口助手验证帧格式**
   - 单板发帧，PC 上用串口助手抓包，确认帧头/帧尾/校验正确

3. **连线联调**
   - 先测心跳互通（双方显示 [OK]）
   - 再测断线检测（拔掉一根线，1s 内显示 [FAIL]）
   - 最后测联动（K1 长按控制 D2，K2 切换影响 D1 呼吸频率）
